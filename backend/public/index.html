API de Facturación y Clientes

Bienvenido a la documentación oficial de la API.

Antes de comenzar a realizar pruebas, necesitas configurar tu entorno de desarrollo.

Autenticación de Pruebas

Para consumir los endpoints protegidos, puedes generar un JSON Web Token (JWT) temporal (24h):

Endpoint de Token https://ad-modulo-facturacion.onrender.com/auth/test-token

Ejemplo de uso

curl -X POST https://ad-modulo-facturacion.onrender.com/auth/test-token \
  -H "Content-Type: application/json"

Entorno de Pruebas

Si prefieres armar tus consultas visualmente y probar los resolvers en tiempo real:

Apollo Sandbox: https://ad-modulo-facturacion.onrender.com/graphql

Nota: Añade el token generado en las cabeceras:

{
  "Authorization": "Bearer TU_TOKEN"
}

Types

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Example
"4"
Types

Int

Description

The Int scalar type represents non-fractional signed whole numeric values. Int can represent values between -(2^31) and 2^31 - 1.

Example
987
