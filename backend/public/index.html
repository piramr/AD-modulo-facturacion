API de Facturación y Clientes

Documentación oficial de la API. Antes de comenzar a realizar pruebas, necesitas configurar tu entorno de desarrollo.

Autenticación de Pruebas

Para consumir los endpoints protegidos, puedes generar un JSON Web Token (JWT) temporal haciendo una petición GET al siguiente servicio de identidad:

  • Endpoint de Token: https://ad-modulo-facturacion.onrender.com/auth/test-token
  • Ejemplo de uso:
curl -X GET [https://ad-modulo-facturacion.onrender.com/auth/test-token](https://ad-modulo-facturacion.onrender.com/auth/test-token) \
  -H "Content-Type: application/json"

Entorno de Pruebas

Si prefieres armar tus consultas visualmente y probar los resolvers en tiempo real, puedes acceder al entorno interactivo de Apollo Server:

Abrir Apollo Server Sandbox

Nota: Recuerda añadir el token generado en las cabeceras de Apollo Sandbox como {"Authorization": "Bearer TU_TOKEN"}.

  $page: Int,
  $limit: Int
) {
  clientes(
    filter: $filter,
    page: $page,
    limit: $limit
  ) {
    totalCount
    pageInfo {
      ...PageInfoFragment
    }
    items {
      ...ClienteFragment
    }
  }
}

  "data": {

      "totalCount": 123,
      "pageInfo": PageInfo,

    }
  }

FacturasFilter

Description

Criterios de filtrado y búsqueda unificados para la consulta de facturas.

Fields
Input Field Description
estado - String Filtra por el estado actual de la factura. Valores válidos: 'Emitida', 'Anulada', 'Pagada'.
clienteId - String Filtra las facturas pertenecientes a un cliente específico mediante su UUID.
tipoPago - String Filtra por la modalidad de pago de la transacción. Valores esperados: 'Efectivo', 'Crédito'.
search - String Realiza una coincidencia parcial sobre el número de factura.
Example
{
  "estado": "xyz789",
  "clienteId": "abc123",
  "tipoPago": "abc123",
  "search": "xyz789"
}
